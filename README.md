# Investigative Operational Security Protocol for OSINT Investigators

A practical operational security framework for conducting open source intelligence investigations safely, consistently, and responsibly.

This template was created as part of the **OSINT Foundations program from OSINT Academy while working toward the OSC credential** and is shared as a free resource for investigators who want to build repeatable investigative workflows.

---

## What This Template Covers

This protocol provides guidance on:

- Client confidentiality and secure communications
- Investigative environment separation
- Managed attribution practices
- Cyber hygiene and system security
- Personal exposure monitoring
- Evidence preservation and documentation
- Attribution risk assessment
- Incident response and escalation contacts

The goal is to help investigators protect themselves, protect their clients, and maintain the integrity of investigative findings.

## How to Use This Template

1. Review the protocol and adapt it to your investigative environment.
2. Customize sections such as escalation contacts, incident response procedures, and documentation practices.
3. Use the pre-investigation checklist before beginning new cases.
4. Update the protocol as your investigative methods and tools evolve.

---

## Important Notes

This template is intended for **lawful open source investigations conducted by civilian investigators**.  
It does not support hacking, unauthorized access to systems, or misrepresentation to obtain restricted information.

Investigators should always ensure their activities comply with applicable laws and professional guidelines.
